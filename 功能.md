## 一、`debug_elements.py` — 元素采集工具

### 核心功能

| 功能                    | 说明                               |
| ----------------------- | ---------------------------------- |
| **F1 拾取模式**         | 点击页面元素，自动采集所有属性信息 |
| **F2 悬停高亮**         | 鼠标移动时高亮显示元素，辅助观察   |
| **F3 设置步骤名**       | 给当前步骤命名，标记在采集数据里   |
| **F4 退出**             | 关闭浏览器                         |
| **Esc 暂停**            | 关闭拾取面板                       |
| **Ctrl+S 保存**         | 把所有采集数据写成 JSON 文件       |
| **跨页面/跨标签继承**   | 无论跳转多少页面，采集数据全部保留 |
| **Python 后台直写文件** | 不走浏览器下载，直接写到项目目录   |

### 采集的字段（每条元素记录）

| 字段                            | 说明                              |
| ------------------------------- | --------------------------------- |
| `time`                          | 采集时间                          |
| `step_name`                     | 步骤名称                          |
| `page_url`                      | 当前页面 URL                      |
| `page_title`                    | 当前页面标题                      |
| `selectors.tag`                 | 标签名                            |
| `selectors.id`                  | 元素 ID                           |
| `selectors.class`               | 元素 class                        |
| `selectors.text`                | 元素文本内容                      |
| `selectors.inner_text`          | 元素内部文本                      |
| `selectors.placeholder`         | placeholder 属性                  |
| `selectors.type`                | type 属性                         |
| `selectors.name`                | name 属性                         |
| `selectors.role`                | role 属性                         |
| `selectors.aria-label`          | aria-label 属性                   |
| `selectors.data-*`              | 所有 data- 属性字典               |
| `selectors.css_selector`        | CSS 选择器                        |
| `selectors.xpath_short`         | 短 XPath                          |
| `selectors.xpath_full`          | 完整 XPath（从 /html 开始）       |
| `selectors.playwright_selector` | Playwright 推荐选择器             |
| `selectors.rect`                | 元素位置和大小                    |
| **`ancestors`**                 | **向上 8 层的稳定祖先链（新增）** |

### `ancestors` 字段内容

| 字段                   | 说明                     |
| ---------------------- | ------------------------ |
| `tag`                  | 祖先标签名               |
| `id`                   | 祖先 ID                  |
| `class`                | 祖先 class               |
| `text`                 | 祖先文本（截取 50 字符） |
| `role`                 | 祖先 role 属性           |
| `aria_label`           | 祖先 aria-label          |
| `depth`                | 距离目标元素的层级       |
| **`is_stable_anchor`** | **是否被判定为稳定锚点** |

### 稳定锚点判定规则

| 条件                                        | 说明            |
| ------------------------------------------- | --------------- |
| 有稳定 id（不含 `el-id-`、不含 `:`）        | 业务固定 ID     |
| `role="dialog"` 或 `role="alertdialog"`     | 弹窗/对话框     |
| class 包含 `el-dialog`                      | Element UI 弹窗 |
| `tag="form"` 或 `role="form"`               | 表单区域        |
| `id="yy-table"` 或 `id="yy-layout-content"` | 领星业务区域    |
| 有 `aria-label` 属性                        | 无障碍标签      |

## 二、`selector_cleaner.py` — 独立清洗模块

### 核心功能

| 功能                           | 说明                                  |
| ------------------------------ | ------------------------------------- |
| **命令行独立运行**             | `python selector_cleaner.py xxx.json` |
| **--inplace 模式**             | 直接修改原文件，新增 `cleaned` 字段   |
| **读取 selectors + ancestors** | 读目标元素信息 + 祖先链信息           |
| **输出 cleaned 字段**          | 新增字段，不破坏原始数据              |

### 清洗规则清单（10 类）

| 规则类型                  | 具体内容                                                        |
| ------------------------- | --------------------------------------------------------------- |
| **1. 动态 ID 识别**       | `el-id-`、`:` (Vue scoped)、纯数字、UUID、`random`、`rc-`       |
| **2. 动态 class 识别**    | `data-v-`、`css-` 哈希、`sc-` (styled-components)、`_` 哈希缩写 |
| **3. 布局/工具类过滤**    | `flex`、`pt-8`、`justify-between`、`ml-auto` 等 30+ 个          |
| **4. 状态类过滤**         | `is-active`、`is-expanded`、`is-focus` 等 8 个                  |
| **5. 测试属性最高优先级** | `data-testid`、`data-cy`、`data-test`、`data-qa`                |
| **6. 埋点属性第二梯队**   | `data-sensors-click`、`data-track`                              |
| **7. 稳定 class 白名单**  | `tree__item`、`add-btn`、`el-button`、`icon-` 等 14 个          |
| **8. 祖先锚定策略生成**   | 用 `ancestors` 生成 dialog/id/aria-label 等锚定版本             |
| **9. 可靠性评级**         | 有锚定→high，有文本→high，有稳定id→medium，其它→low             |
| **10. 过滤记录输出**      | `filtered_out` 记录被丢弃的 id 和 class                         |

### 策略优先级（从高到低）

| 优先级 | 策略类型           | 示例                                  |
| ------ | ------------------ | ------------------------------------- |
| **1**  | 锚定 + 测试属性    | `role=dialog 内 data-testid="submit"` |
| **2**  | 锚定 + 文本        | `role=dialog 内 text="确定"`          |
| **3**  | 锚定 + placeholder | `role=dialog 内 placeholder="请输入"` |
| **4**  | 测试属性           | `data-testid="submit"`                |
| **5**  | 文本               | `text="手动添加"`                     |
| **6**  | aria-label         | `aria-label="关闭"`                   |
| **7**  | placeholder        | `placeholder="请输入"`                |
| **8**  | 埋点属性           | `data-sensors-click="手动添加"`       |
| **9**  | title              | `title="刷新"`                        |
| **10** | name               | `name="username"`                     |
| **11** | role + 名称        | `button[name="提交"]`                 |
| **12** | 稳定 ID            | `yy-table`                            |
| **13** | 稳定 class         | `button.el-button`                    |
| **14** | type 属性          | `input[type="text"]`                  |

### `cleaned` 字段输出结构

```json
{
    "cleaned": {
        "strategies": [
            // 锚定策略排最前面
            {"type": "anchored", "anchor": {...}, "target": {...}, "description": "..."},
            // 自身策略排后面
            {"type": "placeholder", "value": "请输入"},
            {"type": "css", "value": "input[type='text']"}
        ],
        "best_strategy": {...},
        "reliability": "high",
        "filtered_out": {"id": "el-id-9730-91", "classes": ["data-v-abc"]},
        "anchors_available": ["dialog", "yy-layout-content"]
    }
}
```

## 三、完整工作流

```
1. python debug_elements.py
   → 打开浏览器，按 F1 采集元素
   → Ctrl+S 保存 → lingxing_elements.json
         ↓
2. python selector_cleaner.py lingxing_elements.json --inplace
   → 读取 selectors + ancestors
   → 过滤动态 id/class，提取稳定特征
   → 生成锚定策略 + 自身策略
   → 写入 cleaned 字段
         ↓
3. auto_lingxing.py 读取 cleaned.strategies
   → robust_locate 按顺序尝试
   → 优先用锚定策略（弹窗内），再回退自身策略
   → 精准定位，不再报 strict mode violation
```
