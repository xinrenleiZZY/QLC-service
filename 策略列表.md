# 每条元素记录会新增一个 `cleaned` 字段，完整结构如下：

## `cleaned` 字段详解

### `strategies` — 策略列表

| 策略类型      | 结构                                                                | 何时生成               |
| ------------- | ------------------------------------------------------------------- | ---------------------- |
| `anchored`    | `{type, anchor: {type, value}, target: {type, value}, description}` | 有稳定祖先时，排在前面 |
| `text`        | `{type: "text", value: "xxx"}`                                      | 元素有可见文本         |
| `placeholder` | `{type: "placeholder", value: "xxx"}`                               | 输入框有 placeholder   |
| `aria_label`  | `{type: "aria_label", value: "xxx"}`                                | 有 aria-label          |
| `title`       | `{type: "title", value: "xxx"}`                                     | 有 title 属性          |
| `name`        | `{type: "name", value: "xxx"}`                                      | 有 name 属性           |
| `role`        | `{type: "role", value: 'button[name="xxx"]'}`                       | 有 role 属性           |
| `id`          | `{type: "id", value: "xxx"}`                                        | 有稳定 ID（非动态）    |
| `css`         | `{type: "css", value: "xxx"}`                                       | 有稳定 class 组合      |
| `attribute`   | `{type: "attribute", value: '[data-testid="xxx"]'}`                 | 有测试属性或埋点属性   |

### `anchor` 锚点类型

| 锚点类型     | 来源                      | 示例                                           |
| ------------ | ------------------------- | ---------------------------------------------- |
| `role`       | 祖先 `role="dialog"`      | `{"type": "role", "value": "dialog"}`          |
| `id`         | 祖先稳定 ID               | `{"type": "id", "value": "yy-layout-content"}` |
| `aria_label` | 祖先 `aria-label`         | `{"type": "aria_label", "value": "新建词库"}`  |
| `css`        | 祖先 class 含 `el-dialog` | `{"type": "css", "value": ".el-dialog"}`       |

### `reliability` — 可靠性评级

| 值         | 条件                                                   |
| ---------- | ------------------------------------------------------ |
| `"high"`   | 有测试属性 / 有锚定策略 / 第一策略是文本或 placeholder |
| `"medium"` | 有稳定 ID 或稳定 class，但无文本                       |
| `"low"`    | 以上都不满足，只能用 type 属性或兜底 XPath             |

### `filtered_out` — 被过滤的信息

| 字段      | 说明                                                 |
| --------- | ---------------------------------------------------- |
| `id`      | 被丢弃的动态 ID（如 `el-id-9730-91`），无则为 `null` |
| `classes` | 被丢弃的动态 class 列表（如 `data-v-xxx`）           |

### `anchors_available` — 可用锚点

稳定祖先的摘要列表，方便人工查看：

```json
["yy-layout-content", "app"]
```

对应 `ancestors` 里 `is_stable_anchor: true` 的那些。

## 针对你这个具体例子的分析

| 字段          | 值                         | 说明                                           |
| ------------- | -------------------------- | ---------------------------------------------- |
| `tag`         | `div`                      | 一个 div 容器                                  |
| `text`        | 超长（含数字和%）          | 超过 80 字符，**不生成文本策略**               |
| `placeholder` | null                       | 无                                             |
| `id`          | null                       | 无                                             |
| `class`       | `dashboard-chunk flex-col` | `dashboard-chunk` 不在白名单，但通过了动态检查 |
| `data-*`      | 两个 `data-v-*`            | Vue scoped，**被丢弃**                         |
| `ancestors`   | 2 个稳定锚点               | `yy-layout-content` 和 `app`                   |

**最终只有 1 条自身策略 + 2 条锚定策略**，总共 3 条。这个元素没有文本、没有 id、没有 placeholder，所以策略比较少，但通过锚定保证了可靠性为 `high`。
