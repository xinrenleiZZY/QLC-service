# Skill: 领星ERP批量创建词库
# 该文件为skill创建规范，只需要替换skill名字，路径，操作步骤即可
## 描述

自动登录领星ERP，进入广告管理 → 词库管理，批量创建关键词词库。基于 Playwright(主) + CDP/Pyppeteer(辅) 双引擎浏览器自动化，支持失败重试、热替换模块和人工介入。

## 第一步：询问是否需要修改 vars.yaml

文件路径：`e:\ZY2026\可视化设计自动化脚本V10 - 批量上传关键词\vars.yaml`

所有配置都在这一个文件里，用户说人话指令，你改对应字段即可。

### 登录凭证

| 用户可以说                      | 你要做的事                       |
| ------------------------------- | -------------------------------- |
| "把账号改成 xxx"                | 修改 `ACCOUNT` 的值              |
| "把密码改成 xxx"                | 修改 `PASSWORD` 的值             |
| "换账号登录，账号 xxx 密码 xxx" | 同时修改 `ACCOUNT` 和 `PASSWORD` |

### 店铺筛选

| 用户可以说      | 你要做的事                       |
| --------------- | -------------------------------- |
| "搜索美国站"    | 修改 `SEARCH_KEYWORD` 为 `"US"`  |
| "搜索加拿大站"  | 修改 `SEARCH_KEYWORD` 为 `"CA"`  |
| "搜索 xxx 店铺" | 修改 `SEARCH_KEYWORD` 为对应的值 |

### ASIN 列表（要上传的商品）

| 用户可以说                      | 你要做的事                                                  |
| ------------------------------- | ----------------------------------------------------------- |
| "ASIN 换成这几个：B0xxx、B0xxx" | 替换 `SEARCH_ASIN` 的多行内容，注意保留 YAML 的 `\|` 块格式 |
| "加一个 ASIN：B0xxx"            | 在 `SEARCH_ASIN` 末尾追加一行                               |
| "去掉 ASIN B0xxx"               | 从 `SEARCH_ASIN` 中删除对应行                               |

示例格式：

```yaml
SEARCH_ASIN: |
  B0DLGJY713
  B0FM1V2VYB
```

### 关键词数据

| 用户可以说                   | 你要做的事                                   |
| ---------------------------- | -------------------------------------------- |
| "关键词输入 phrase 和 board" | 修改 `INPUT_VALUE_KEY` 为 `"phrase\nboard"`  |
| "phrase 输入 1、2、3"        | 修改 `INPUT_VALUE_KEY_VALUE1` 为 `"1\n2\n3"` |
| "board 输入 4、5、6"         | 修改 `INPUT_VALUE_KEY_VALUE2` 为 `"4\n5\n6"` |
| "关键词数据都换一下..."      | 根据用户描述对应修改三个字段                 |

### 其他配置

| 用户可以说         | 你要做的事                          |
| ------------------ | ----------------------------------- |
| "词库名字改成 xxx" | 修改 `INPUT_VALUE` 的值             |
| "国家选美国"       | 修改 `INPUT_VALUE_US` 为 `"美国"`   |
| "国家选加拿大"     | 修改 `INPUT_VALUE_US` 为 `"加拿大"` |

## 第二步：运行脚本

先询问用户是否需要修改 vars.yaml，用户确认可以运行了，执行：

```powershell
start "领星自动化" e:\ZY2026\可视化设计自动化脚本V10 - 批量上传关键词\run_interactive.bat
```

这会弹出一个新的 CMD 窗口，执行过程中如果出错会有交互菜单（重试/跳过/人工检查等），用户直接在 CMD 窗口里操作即可。

### 完整对话示例

**用户**：帮我跑一趟领星批量创建词库，ASIN 换成 B0123456789 和 B0987654321，词库名叫"测试0506"

**你应做的事**：

1. 读取 vars.yaml
2. 修改 `SEARCH_ASIN` → `"B0123456789\nB0987654321"`
3. 修改 `INPUT_VALUE` → `"测试0506"`
4. 保存 vars.yaml
5. 问用户："已改好，其他参数（账号密码、筛选词等）保持不变。要现在运行吗？"
6. 用户确认后，执行 `start` 命令弹出 CMD 窗口

**用户**：改下密码，原来是 123456 改成 abcdef

**你应做的事**：

1. 读取 vars.yaml
2. 修改 `PASSWORD` → `"abcdef"`
3. 保存 vars.yaml
4. 回复："密码已更新"

## 用户交互说明

执行过程中弹出的 CMD 窗口里会出现以下交互菜单，用户直接在 CMD 窗口中按键操作即可：

### 模块失败时

```
  r — 重试失败的元素
  a — 全部重新执行此模块
  h — 人工检查（浏览器保持，操作后按回车）
  n — 用新 JSON 替换此模块（热替换）
  i — 在此模块后插入一个新步骤
  s — 跳过失败，继续下一个模块
  q — 终止整个流程
```

### 模块成功时（10秒无操作自动继续）

```
  i — 在此后插入一个新步骤
  r — 回到重试菜单
  c — 继续执行下一个模块
```

### 流程结束时

```
  q — 退出并关闭浏览器
  k — 保持浏览器打开（手动关闭）
```

| 问题               | 你该怎么做                                                         |
| ------------------ | ------------------------------------------------------------------ |

| 用户说配置改乱了   | 读取 vars.yaml 按上面的格式输出给用户确认，让用户重新说            |
| 用户想临时跳过某步 | 告诉用户在 CMD 窗口里按 `s` 跳过                                   |
