## 完整流程

### 第 1 步：启动浏览器实例

**文件**: `start_browser.py` / `session.py`

```bash
python start_browser.py
```

- 用 Playwright 启动 Chrome（持久上下文 `playwright_profile`）
- **关键**: 带 `--remote-debugging-port=18800` 参数，打开 CDP 调试端口
- 浏览器保持打开（`asyncio.sleep(36000)`），不关闭
- 可用 `http://127.0.0.1:18800` 连接

> 如果已有浏览器在 18800 端口运行，`session.py` 会自动检测并**复用**（通过 `_detect_existing()`），不会重复启动。

---

### 第 2 步：随时采集元素（随时接入、随时退出）

**文件**: `debug_elements-V10.py`

```bash
python debug_elements-V10.py --cdp http://127.0.0.1:18800
```

- 连接到**已有**浏览器实例（不重新启动）
- 用 F1/F2/F3/F4 快捷键采集元素：
  - **F1**: 采集当前选中元素（带祖先链 `ancestors`）
  - **F2**: 切换悬停模式
  - **F3**: 修改步骤名
  - **F4**: 保存 JSON
- 输出: `lingxing_elements_V10_*.json`（V10 格式：多个 `{metadata, elements}` 包裹体）

**特点**: 浏览器可以一直开着，采集工具可以随时连接/断开，互不影响。

---

### 第 3 步：清洗 JSON

**文件**: `selector_cleaner.py`

```bash
python selector_cleaner.py lingxing_elements_V10_新建词库.json
```

- 读取 V10 原始 JSON（含 `ancestors`、原始 `selectors`）
- 清洗过程：
  - 提取 `elements` 数组（兼容 V10 包裹格式）
  - 过滤动态 class/id（如 `el-id-123`、`data-v-xxx`）
  - 保留稳定 class 片段（如 `el-button`、`yy-table`）
  - 生成多策略定位:
    1. **text** 文本匹配
    2. **placeholder** 占位符匹配
    3. **anchored** 锚定策略（根据父级/兄弟定位）
    4. **selector** CSS 选择器
    5. **xpath** XPath 定位
  - 写入 `cleaned.strategies` 字段
- 输出: `lingxing_elements_*_cleaned.json`

---

### 第 4 步：配置动作 YAML

**文件**: `configure_actions.py` / 手动编辑

```bash
python configure_actions.py lingxing_elements_V10_新建词库_cleaned.json
```

生成 `actions_config_*.yaml`，定义每个元素的行为：

```yaml
actions:
  - index: 4
    action:
      type: click # click / fill / select / hover
      on_not_found: skip # fail=必须找到 / skip=可跳过
      wait_for_element: true # 是否等待元素出现
      wait_before: 0.5 # 操作前等待(秒)
      wait_after: 2.0 # 操作后等待(秒)
      wait_strategy: navigation # timeout / navigation / element_appear
      value: "${INPUT_VALUE}" # fill 时的输入值（支持变量）
      clear_first: true # fill 前是否清空
```

**变量系统**: `vars.yaml` 中的 `${变量名}` 可在 action value 中引用。

---

### 第 5 步：编排启动

**文件**: `orchestrator.py` + `flow_*.yaml`

```bash
python orchestrator.py flow_lingxing_elements_V10_领星批量创建词库.yaml
```

**Flow YAML** 定义流程：

```yaml
name: lingxing_elements_V10_领星批量创建词库
modules:
  - name: lingxing_elements_V10_首页
    json: lingxing_elements_V10_首页_cleaned.json
  - name: lingxing_elements_V10_新建词库
    json: lingxing_elements_V10_新建词库_cleaned.json
  - name: lingxing_elements_V10_新建词库_5
    json: lingxing_elements_V10_新建词库_5_cleaned.json
```

编排器执行流程：

```
Orchestrator
  │
  ├─ 1. 加载 flow.yaml → 模块列表
  ├─ 2. BrowserSession.create() → 复用/启动浏览器
  ├─ 3. 依次执行每个模块:
  │      ┌─────────────────────────────────────┐
  │      │ ModuleRunner.run(json, yaml)        │
  │      │   ├─ 加载 JSON（兼容3种格式）         │
  │      │   ├─ 加载 YAML（动作配置）            │
  │      │   ├─ 按 step_name 分组               │
  │      │   ├─ 按组执行（每组一个区域）           │
  │      │   │   └─ StepRunner.run(每个元素)     │
  │      │   │       ├─ robust_locate(多策略)    │
  │      │   │       ├─ 按 action.type 执行动作  │
  │      │   │       └─ 返回 SUCCESS/SKIP/FAIL  │
  │      │   └─ 返回 RunResult                  │
  │      └─────────────────────────────────────┘
  │         │
  │         └─ 如果失败 → 交互菜单:
  │              r=重试 / a=全部重来 / h=人工介入
  │              n=热替换JSON / i=插入新步骤
  │              s=跳过 / q=终止
  │
  ├─ 4. 最终报告
  └─ 5. 退出时关闭浏览器
```

## 核心模块详解

| 层         | 文件                    | 职责                                                         |
| ---------- | ----------------------- | ------------------------------------------------------------ |
| **采集**   | `debug_elements-V10.py` | F1-F4 采集页面元素，输出 V10 JSON                            |
| **清洗**   | `selector_cleaner.py`   | 原始 JSON → 多策略定位 `cleaned`                             |
| **配置**   | `configure_actions.py`  | 自动生成动作 YAML                                            |
| **浏览器** | `session.py`            | 浏览器长连接，CDP + OCR 引擎管理                             |
| **单模块** | `module_runner.py`      | 执行一个 JSON 的所有元素                                     |
| **编排**   | `orchestrator.py`       | 多模块流程管理 + 失败处理                                    |
| **引擎**   | `advanced_runner.py`    | 状态机、双引擎定位、重试策略                                 |
| **定位**   | `core/selector.py`      | 多策略定位核心（text/placeholder/anchor/xpath/selector/OCR） |
| **辅助**   | `start_browser.py`      | 单独启动浏览器实例                                           |

### 元素定位策略链（`robust_locate`）

```
1. text 匹配          ── 用元素的文本内容定位
2. placeholder 匹配   ── 用 input 的 placeholder 定位
3. anchored 锚定      ── 根据附近稳定元素定位
4. selector CSS       ── 用 CSS 选择器定位
5. xpath              ── 用 XPath 定位
6. CDP 深度搜索       ── 通过 CDP 协议在 DOM 中深度搜索
7. OCR 文字识别       ── 截图后用 OCR 识别文字位置（最后手段）
```

## 典型工作流（从零到执行）

```
bash 1
├─ python start_browser.py                                # 一直开着浏览器
│
bash 2 (随时)
├─ python debug_elements-V10.py --cdp http://127.0.0.1:18800  # F1采集→F4保存
│
bash 3
├─ python selector_cleaner.py lingxing_*.json              # 清洗
│
bash 4
├─ python configure_actions.py lingxing_*_cleaned.json     # 生成动作配置
│  (或手动编辑 YAML 调整 on_not_found、type 等)
│
bash 5
├─ python orchetrator.py flow_*.yaml                      # 一键执行
│
│  执行中失败时:
│  → 人工介入 / 热替换 JSON / 插入新步骤
│  → 或重试 / 跳过
│
└─ 执行完毕 → 关闭浏览器
```

## Action YAML（单元素动作配置）

actions:

- index: 4 # 对应 JSON 中元素的索引
  step_name: "步骤名" # 标识，仅注释作用
  action:

  # ── 动作类型 ──

  type: click # click / fill / select / hover # wait_visible / check_exists # check_visible / scroll_to # screenshot / locate_parent

  # ── 失败处理 ──

  on_not_found: skip # fail = 必须找到，找不到就终止 # skip = 找不到就跳过，继续下一个

  # ── 等待行为 ──

  wait_for_element: true # 是否先等待元素出现再定位（默认 false）
  wait_timeout: 10 # 等待超时秒数（默认 10）
  wait_before: 0.5 # 操作前等待秒数（默认 0.3）
  wait_after: 2.0 # 操作后等待秒数（默认 0.5）

  # ── 操作后等待策略 ──

  wait_strategy: timeout # timeout = 固定等 wait_after 秒 # navigation = 等待页面加载完成 # element_appear = 等待 DOM 就绪

  # ── fill 相关 ──

  value: "${INPUT_VALUE}" # fill 的输入值 / select 的选项值 # 支持 ${变量名} 引用 vars.yaml
  clear_first: true # fill 前是否先清空（默认 true）

  # ── screenshot 相关 ──

  screenshot_name: "登录页" # 截图文件名
