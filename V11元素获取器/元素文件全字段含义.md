# 元素文件 `*_cleaned.json` 全字段含义

## 顶层结构

```json
{
  "time": "2026-06-10T02:09:41.614Z",
  "step_name": "登录",
  "page_url": "https://huizhixin.lingxing.com/erp/home",
  "page_title": "领星ERP - 跨境电商管理系统",
  "selectors": { ... },
  "ancestors": [ ... ],
  "cleaned": { ... }
```

| 字段 | 类型 | 含义 | 来源 |
|---|---|---|---|
| `time` | string | 采集时间戳 (ISO 8601) | `debug_elements-V11.py` 采集时自动记录 |
| `step_name` | string | 步骤名，同一步骤的元素按此分组依次执行 | 用户在 F3 面板输入 |
| `page_url` | string | 采集时的页面 URL | 浏览器自动获取 |
| `page_title` | string | 采集时的页面标题 | 浏览器自动获取 |
| `selectors` | object | 从页面提取的所有原始选择器信息 | 采集器 JS 自动提取 |
| `ancestors` | array | 祖先元素链（用于锚定定位） | 采集器 JS 自动提取 |
| `cleaned` | object | 清洗后生成的定位策略（核心） | `selector_cleaner.clean_element()` 生成 |

---

## `selectors` — 原始选择器

```json
{
  "tag": "button",
  "id": "__pickerExitConfirm",
  "class": null,
  "text": "确定退出",
  "placeholder": null,
  "type": null,
  "name": null,
  "data-*": null,
  "css_selector": "#__pickerExitConfirm",
  "xpath_short": "//*[@id=\"__pickerExitConfirm\"]",
  "xpath_full": "/html[1]/body[1]/div[6]/div[1]/div[3]/button[1]",
  "playwright_selector": "page.locator(\"#__pickerExitConfirm\")",
  "rect": { "x": 863, "y": 460.5, "width": 104, "height": 36 },
  "inner_text": "确定退出"
}
```

| 字段 | 类型 | 含义 |
|---|---|---|
| `tag` | string | HTML 标签名。常见值：`button`, `input`, `div`, `span`, `a`, `textarea`, `select`, `li`, `img` |
| `id` | string or null | HTML `id` 属性。常为动态值，清洗时会判断是否动态，如 `el-id-5062-151` |
| `class` | string or null | HTML `class` 属性（原始值，空格分隔） |
| `text` | string or null | 元素可见文本（最多 100 字），通过 `textContent` 获取 |
| `placeholder` | string or null | `<input>` / `<textarea>` 的 `placeholder` 属性 |
| `type` | string or null | `<input>` 的 `type` 属性。常见值：`text`, `password`, `email`, `checkbox`, `radio`, `submit`, `button` |
| `name` | string or null | HTML `name` 属性，常用于表单提交 |
| `data-*` | object or null | 所有 `data-` 开头的自定义属性，如 `{"data-testid": "login-btn"}` |
| `css_selector` | string | 自动生成的 CSS 选择器路径（优先用 id，无 id 则用 class+nth-child 组合） |
| `xpath_short` | string | 简短的 XPath 表达式（优先用 `@id`） |
| `xpath_full` | string | 从 `<html>` 开始的完整 XPath（包含所有层级索引） |
| `playwright_selector` | string | Playwright 框架的定位代码片段，如 `page.getByText("确定退出")` |
| `rect` | object | 元素在视口中的位置和尺寸。`{ x, y, width, height, top, right, bottom, left }` |
| `inner_text` | string or null | 元素的 `innerText`（最多 80 字），与 `text` 区别在于它会保留换行和空白合并方式 |

## `ancestors` — 祖先元素链

```json
[
  {
    "tag": "div",
    "id": "__pickerExitOverlay",
    "class": null,
    "text": "⏹ 确认退出工具 ...",
    "role": null,
    "aria_label": null,
    "depth": 3,
    "is_stable_anchor": true
  }
]
```

| 字段 | 类型 | 含义 |
|---|---|---|
| `tag` | string | 祖先元素的 HTML 标签名 |
| `id` | string or null | 祖先元素的 `id` 属性 |
| `class` | string or null | 祖先元素的 `class` 属性 |
| `text` | string | 祖先元素的文本内容（截取前 50 字） |
| `role` | string or null | ARIA `role` 属性。常见值：`dialog`, `alertdialog`, `tooltip`, `form`, `navigation` |
| `aria_label` | string or null | ARIA `aria-label` 属性 |
| `depth` | number | 离目标元素的层数。1 = 父元素，2 = 祖父元素，最大 8 |
| `is_stable_anchor` | bool or null | 是否适合作稳定锚点。判断条件：有稳定 id、role 为 dialog、class 含 `el-dialog`、有 aria_label |

### 被标记为稳定锚点的条件

满足任一即标记 `is_stable_anchor: true`：
- `id` 存在且不含动态前缀（如 `el-id-`、`:`）
- `role` 为 `dialog` / `alertdialog`
- 有 `aria_label`
- class 含 `el-dialog`
- 特殊固定 id：`yy-table`, `yy-layout-content`

## `cleaned` — 清洗后定位策略（核心字段）

```json
{
  "strategies": [ ... ],
  "best_strategy": { ... },
  "reliability": "high",
  "filtered_out": { "id": "...", "classes": [...] },
  "anchors_available": [ ... ]
}
```

| 字段 | 类型 | 含义 |
|---|---|---|
| `strategies` | array | 按优先级从高到低排列的定位策略列表。执行引擎按顺序依次尝试，找到即停 |
| `best_strategy` | object or null | `strategies[0]`，引擎认为最可能成功的策略 |
| `reliability` | string | 可靠性评级：`high` / `medium` / `low` |
| `filtered_out` | object | 清洗过程中被判定为动态/无意义而过滤掉的属性 |
| `anchors_available` | array of string | 可用的祖先锚点类型/id 列表（方便调试） |

### `reliability` 判定规则

| 等级 | 判定条件 |
|---|---|
| `high` | 有测试属性（`data-testid` 等）**或** 有锚定策略 **或** 首条策略为 `text` / `aria_label` / `placeholder` |
| `medium` | 有稳定 id 或稳定 class（无文本匹配） |
| `low` | 仅有通用 CSS 选择器（如 `input[type="text"]`），易匹配到多个元素 |

### `filtered_out` 过滤信息

```json
{
  "id": "el-id-5062-151",
  "classes": ["data-v-abcde", "is-active"]
}
```

| 子字段 | 含义 |
|---|---|
| `id` | 被判定为动态值而过滤掉的 id。`null` 表示原始 id 就是稳定的（保留在策略中） |
| `classes` | 被判定为动态/无意义而过滤掉的 class 列表。空数组 `[]` 表示全部保留 |

### `anchors_available` 可用锚点示例

```json
["tooltip", "el-popper-container-5062"]
```

值为祖先的 `role` / `id` / `tag`，用于排查哪些锚点可用。

## `strategies[]` — 策略列表详解

每条策略是一个对象，类型不同则字段不同。

### 类型 1：`anchored` — 锚定定位（最高优先级）

```json
{
  "type": "anchored",
  "anchor": {
    "type": "css",
    "value": "[id^=\"el-popper-container\"]"
  },
  "target": {
    "type": "placeholder",
    "value": "搜索"
  },
  "description": "css=[id^=el-popper-container] 内 placeholder=搜索"
}
```

| 字段 | 含义 |
|---|---|
| `anchor.type` | 锚点定位方式：`id` / `role` / `aria_label` / `css` / `attribute` |
| `anchor.value` | 锚点的值 |
| `target` | 容器内部的定位策略，格式与普通策略一致 |
| `description` | 人类可读的描述文本 |

**执行逻辑**：先定位锚点容器 → 在容器内部再执行 target 策略定位目标元素。

**锚点匹配规则**：

| anchor.type | 匹配方式 |
|---|---|
| `role` | `get_by_role(value)`，取最后一个匹配 |
| `id` | 先 `#精确匹配`，失败则用 `[id^="前缀"]` 前缀匹配 |
| `aria_label` | `[aria-label="value"]` |
| `css` | `locator(value)` |

### 类型 2-10：普通策略

| `type` | 含义 | Playwright 映射 | 示例 value |
|---|---|---|---|
| `text` | 文本精确匹配 | `page.get_by_text(value, exact=True)` | `"确定退出"` |
| `placeholder` | 占位符匹配 | `page.get_by_placeholder(value)` | `"请输入搜索关键词"` |
| `aria_label` | ARIA 标签 | `locator('[aria-label="value"]')` | `"关闭对话框"` |
| `title` | title 属性 | `locator('[title="value"]')` | `"编辑"` |
| `name` | name 属性 | `locator('[name="value"]')` | `"username"` |
| `id` | CSS ID（精确匹配） | `locator('#value')` | `"submit-btn"` |
| `css` | CSS 选择器 | `locator(value)` | `"button.el-button.el-button--primary"` |
| `role` | ARIA role + 名称 | `get_by_role(role, name=...)` 或 `get_by_role(value)` | `'button[name="提交"]'` |
| `attribute` | 自定义属性 | `locator(value)` | `'[data-testid="login-btn"]'` |

### 策略生成优先级

执行引擎按以下顺序尝试，找到第一个可见元素即停止：

```
1.  anchored 锚定           ← 在稳定容器内定位
2.  text 文本匹配            ← 用户可见文本
3.  placeholder 占位符       ← input 的 placeholder
4.  aria_label 无障碍标签    ← 可访问性属性
5.  title 标题属性           ← title 属性
6.  name 名称属性            ← name 属性
7.  id 稳定 ID              ← CSS #id
8.  css 选择器              ← 稳定 class + tag 组合
9.  xpath                   ← 旧式 XPath
10. CDP 深度搜索             ← 穿透 Shadow DOM、iframe
11. OCR 文字识别             ← 截图 + Tesseract（最后一层兜底）
```

## 数据流关系图

```
采集（debug_elements-V11.py）
  │
  ├── selectors      ← JS 从 DOM 中提取所有属性
  ├── ancestors      ← JS 遍历 parentElement 收集
  ├── step_name      ← 用户通过 UI 面板设置
  └── page_url       ← window.location.href
       │
       ▼
清洗（selector_cleaner.clean_element()）
  │
  ├── 过滤动态 id/class
  ├── 筛选可用的祖先锚点
  ├── 按优先级排序生成 strategies
  └── 标记 reliability 评级
       │
       ▼
执行（StepRunner → core/selector.py）
  │
  ├── 按 strategies[] 顺序尝试
  ├── 成功 → 执行动作（click/fill/select）
  └── 全部失败 → OCR 兜底 / 报告失败
```
