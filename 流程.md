## run_custom_steps.py 架构讲解

### 一、整体数据流

```
┌─────────────┐     ┌──────────────┐     ┌──────────┐
│ cleaned.json│     │ actions.yaml │     │ vars.yaml│
│ (定位策略)   │     │ (行为配置)    │     │ (变量值)  │
└──────┬──────┘     └──────┬───────┘     └────┬─────┘
       │                   │                   │
       ▼                   ▼                   ▼
   ┌─────────────────────────────────────────────┐
   │              run() 主函数                      │
   │  1. 加载 JSON → elements[]                   │
   │  2. 加载 YAML → actions_dict{index: action}  │
   │  3. 加载 vars  → vars_dict{ACCOUNT: "...", } │
   │  4. 启动浏览器                                │
   │  5. 遍历每个元素 → 『定位 + 执行』              │
   │  6. 统计结果 → 关闭浏览器                      │
   └─────────────────────────────────────────────┘
```

### 二、核心函数链（一次动作的生命周期）

```
_execute_action(page, strategies, action, vars_dict)
  │
  ├── 1. _robust_locate_with_retry(page, strategies)
  │      └── 循环调用 robust_locate() 最多 5 次，每次间隔 1 秒
  │            └── core/selector.py → 按 strategies 列表依次尝试定位
  │
  ├── 2. sleep(wait_before)          ← 操作前等待
  │
  ├── 3. 按 action.type 执行:
  │      click   → loc.click()
  │      fill    → loc.fill(解析变量后的值)
  │      wait_visible → loc.wait_for()
  │      hover/select → 对应操作
  │
  ├── 4. 操作后等待策略:
  │      timeout    → sleep(wait_after)
  │      navigation → wait_for_load_state("load")
  │      element_appear → wait_for_load_state("domcontentloaded")
  │
  └── 5. 返回 (成功/失败, 消息)
```
