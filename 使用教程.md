## 1. 环境准备

### 安装 Python 依赖

```bash
pip install playwright pyyaml pillow pytesseract
playwright install chromium
```

```bash
pip install pyppeteer
```

```

├── core/                    # 定位引擎
├── *.py                     # 脚本工具
├── *.html                   # 可视化编辑器
```

## 2. 整体工作流

```
启动浏览器 ─→ 采集元素 ─→ 清洗JSON ─→ 配置动作YAML ─→ 编排流程 ─→ 运行
                                         ↑
                                    变量 vars.yaml
```

每一步都是独立的，可以随时进行。浏览器可以一直开着，工具随时连接/断开。

## 3. 一、启动浏览器

### 方式 A：用 `start_browser.py` 启动（推荐）

```bash
python start_browser.py
```

- Chromium 浏览器自动启动（非无头模式）
- CDP 调试端口: **18800**
- 浏览器保持打开，按 `Ctrl+C` 关闭

### 方式 B：让编排器自动启动

```bash
python orchestrator.py flow_xxx.yaml
```

`orchestrator.py` 会自动检测是否有浏览器在 18800 端口运行：

- **有** → 复用已有实例
- **没有** → 自动启动新实例

### 方式 C：OpenClaw 启动的浏览器

框架会自动检测端口 18800 上的 Chrome 实例，支持复用。

## 4. 二、采集元素

### 连接浏览器并采集

```bash
python debug_elements-V10.py --cdp http://127.0.0.1:18800
```

### 快捷键操作

| 按键       | 功能                                          |
| ---------- | --------------------------------------------- |
| **F1**     | 激活/停用拾取模式。激活后点击页面元素自动采集 |
| **F2**     | 激活/停用高亮模式。鼠标移动查看元素信息       |
| **F3**     | 快速定位到步骤名输入框                        |
| **F4**     | 退出工具（浏览器保持运行）                    |
| **Ctrl+S** | 保存采集数据到 JSON 文件                      |
| **ESC**    | 关闭拾取模式                                  |

### 操作步骤

1. 运行 `python debug_elements-V10.py --cdp http://127.0.0.1:18800`
2. 按 **F1** 激活拾取模式
3. 在页面上依次点击需要自动化的元素
4. 在 UI 面板中填写**步骤名**（比如"登录"、"搜索"、"新建词库"）
5. 同一步骤下的元素会被分组执行
6. 全部采集完后按 **F4** → 确认退出 → 自动保存

### 输出

生成 `lingxing_elements_V10_*.json` 文件，包含每个元素的：

- `step_name` — 步骤名（分组依据）
- `page_url` — 当前页面 URL
- `selectors` — 所有选择器（id/class/text/xpath/css）
- `ancestors` — 祖先元素链（用于锚定定位）

### 注意事项

- 同一步骤的元素名要一致（如"搜索结果"下采集 10 个元素都填"搜索结果"）
- 如果元素在弹窗/对话框中，确保弹窗打开时再采集
- 图片/Canvas 元素无法采集选择器，需要用 OCR

## 5. 三、清洗数据

```bash
python selector_cleaner.py lingxing_elements_V10_新建词库.json
```

自动生成 `lingxing_elements_V10_新建词库_cleaned.json`

### 清洗做了什么

| 处理           | 说明                                                          |
| -------------- | ------------------------------------------------------------- |
| 过滤动态 ID    | `el-id-5062-151`、`el-popper-container-5062` 等动态 ID 被过滤 |
| 过滤动态 class | `data-v-xxx`、`sc-xxx` 等无用 class 被移除                    |
| 保留稳定 class | `el-button`、`yy-table` 等有语义的 class 保留                 |
| 生成多策略     | text / placeholder / anchored / css / xpath 多个策略          |
| 锚定策略       | 根据祖先元素生成"在 XX 对话框内找 YY 按钮"的策略              |
| 标记可靠性     | `high` / `medium` / `low` 三个等级                            |

### 策略优先级

```
1. anchored 锚定     ← 最高优先级（在特定容器内定位）
2. text 文本匹配      ← 用文本定位
3. placeholder       ← input 的 placeholder
4. css 选择器         ← CSS 选择器
5. xpath             ← XPath 定位
```

### 多文件批量清洗

```bash
# Windows PowerShell
Get-ChildItem *.json -Exclude "*_cleaned*" | ForEach-Object {
    python selector_cleaner.py $_.Name
}
```

## 6. 四、配置动作

### 方式 1：CLI 配置（`configure_actions.py`）

```bash
python configure_actions.py lingxing_elements_V10_新建词库_cleaned.json
```

逐一询问每个元素的：

- 操作类型（点击/填写/检测/悬浮/...）
- 找不到时行为（跳过/终止）
- 是否需要等待元素出现
- 填写内容（引用变量）
- 定时参数

### 方式 2：可视化编辑器（推荐）

直接用浏览器打开 `yaml_editor.html`，界面操作，详见[第十章](#10-十可视化编辑器)。

### 自动生成默认配置

如果不配置动作，`module_runner.py` 会根据元素标签自动推断：

| 标签                     | 默认动作                                        |
| ------------------------ | ----------------------------------------------- |
| `<input>` / `<textarea>` | `fill` — 填入 `${INPUT_VALUE}`                  |
| `<button>` / `<a>`       | `click`（含"登录/提交/确认"关键词则等页面加载） |
| `<select>`               | `select`                                        |
| 步骤名含"父元素"         | `locate_parent` — 设为容器                      |
| 步骤名含"是否可见"       | `check_exists` — 检测存在                       |
| 其他                     | `click`                                         |

### 输出

生成 `actions_config_lingxing_elements_V10_新建词库.yaml`

## 7. 五、变量配置

编辑 `vars.yaml` 文件：

```yaml
# 登录凭证
ACCOUNT: "your_account"
PASSWORD: "your_password"

# 搜索关键词
SEARCH_KEYWORD: "US"

# 通用输入值
INPUT_VALUE: "测试内容"

# ASIN 列表（多行）
SEARCH_ASIN: |
  B0DLGJY713
  B0FM1V2VYB
```

### 在 YAML 中引用变量

```yaml
- index: 5
  action:
    type: fill
    value: "${ACCOUNT}" # → 实际填入 "your_account"
```

编辑 `vars.yaml` 后保存即可，执行时自动加载。

### 可视化编辑

在 `yaml_editor.html` 中切换到"变量"标签页，可直接编辑变量值。

## 8. 六、编辑流程 YAML

### 基本结构

```yaml
name: 流程名称
modules:
  - name: 首页
    json: lingxing_elements_V10_首页_cleaned.json
  - name: 词库
    json: lingxing_elements_V10_词库_cleaned.json
    yaml: actions_config_lingxing_elements_V10_词库.yaml # 可选
```

### 文件名命名规则

| 文件        | 命名规则                            | 示例                               |
| ----------- | ----------------------------------- | ---------------------------------- |
| 原始 JSON   | `lingxing_elements_V10_步骤名.json` | `..._新建词库.json`                |
| 清洗后 JSON | `*_cleaned.json`                    | `..._新建词库_cleaned.json`        |
| 动作配置    | `actions_config_{stem}.yaml`        | `actions_config_..._新建词库.yaml` |

### 多个文件组合

如果一个页面有多个步骤，可以拆成多个 JSON：

```yaml
modules:
  - name: lingxing_elements_V10_新建词库
    json: lingxing_elements_V10_新建词库_cleaned.json
  - name: lingxing_elements_V10_新建词库_2
    json: lingxing_elements_V10_新建词库_2_cleaned.json
```

## 9. 七、运行编排

```bash
python orchestrator.py flow_lingxing_elements_V10_领星批量创建词库.yaml
```

### 执行过程

```
1. 加载流程定义
2. 检测/启动浏览器（复用端口 18800）
3. 按顺序执行每个模块
   ├─ 加载 JSON + YAML
   ├─ 按 step_name 分组
   ├─ 每个区域依次执行
   │   ├─ 导航到目标 URL
   │   ├─ 定位策略（多策略尝试）
   │   ├─ 执行动作（click/fill/select）
   │   └─ 等待策略（timeout/navigation）
   └─ 返回结果
4. 模块成功 → 10 秒无操作自动继续
5. 模块失败 → 交互菜单
6. 最终报告
7. 退出时选择是否关闭浏览器
```

### 运行多个流程

可以拆成多个 flow.yaml，用一个入口 flow 引用：

```yaml
name: 完整流程
pre_flows:
  - flow_检查登录态.yaml # 先检查登录
modules:
  - name: lingxing_elements_V10_首页
    json: ...
```

## 10. 八、运行时处理

### 模块成功后

```
✅ 模块 [xxx] 执行完毕
  i — 在此后插入一个新步骤
  r — 回到模块重试菜单
  c — 继续执行下一个模块
  (10秒内无输入自动继续)
```

- 默认 10 秒自动继续，**无需人工干预**
- 可以插入新模块到流程中

### 模块失败时

```
==================================================
  模块 [xxx] 有 N 个元素失败
  失败索引: [4, 7]
==================================================
  r — 重试失败的 N 个元素
  a — 全部重新执行此模块
  h — 人工检查（浏览器保持，操作后回车）
  n — 用新 JSON 替换此模块（热替换）
  i — 在此模块后插入一个新步骤
  s — 跳过失败，继续下一个模块
  q — 终止整个流程
  ──────────────────────────────
  选择:
```

| 选项  | 适用场景                                     |
| ----- | -------------------------------------------- |
| **r** | 可能是网络延迟，重试失败的元素               |
| **a** | 整个模块状态不对，全部重来                   |
| **h** | 需要手动操作（处理验证码等），操作后回车继续 |
| **n** | 重新采集了元素，用新 JSON 替换               |
| **i** | 发现漏了步骤，插入一个新模块                 |
| **s** | 非关键步骤，跳过继续                         |
| **q** | 无法继续，终止                               |

## 11. 九、Flow YAML 高级配置

### 条件跳转

```yaml
modules:
  - name: 检测是否处于登录态
    json: ...
    yaml: ...
    # 检测失败 → 跳转到登录模块
    on_failure:
      action: goto
      target: lingxing_elements_V10_登录
  - name: lingxing_elements_V10_退出登录
    json: ...
  - name: lingxing_elements_V10_登录
    json: ...
```

### 支持的跳转动作

| 动作         | 说明                          | 参数             |
| ------------ | ----------------------------- | ---------------- |
| `goto`       | 跳转到指定模块                | `target: 模块名` |
| `skip_next`  | 跳过接下来 N 个模块           | `count: N`       |
| `abort`      | 终止流程                      | —                |
| `retry_once` | 自动重试一次（仅 on_failure） | —                |

### 前置流程

```yaml
name: 领星批量创建词库

pre_flows:
  - flow_检查登录态.yaml # 先执行检查登录态流程

modules:
  - name: lingxing_elements_V10_首页
    json: ...
```

前置流程会在主流程前执行，所有模块合并到一起运行。

### 模块内元素动作参数

| 参数               | 说明             | 可选值                                                                                                                    |
| ------------------ | ---------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `type`             | 操作类型         | `click` / `fill` / `select` / `hover` / `check_exists` / `check_visible` / `wait_visible` / `scroll_to` / `locate_parent` |
| `on_not_found`     | 找不到时         | `fail`（终止） / `skip`（跳过）                                                                                           |
| `wait_before`      | 操作前等待（秒） | 数值，默认 0.5                                                                                                            |
| `wait_after`       | 操作后等待（秒） | 数值，默认 0.5                                                                                                            |
| `wait_strategy`    | 等待策略         | `timeout` / `navigation` / `element_appear`                                                                               |
| `wait_for_element` | 是否等待元素出现 | `true` / `false`                                                                                                          |
| `value`            | fill 的值        | 字符串或 `${变量名}`                                                                                                      |
| `clear_first`      | fill 前清空      | `true` / `false`                                                                                                          |
| `click_first`      | 先点击再输入     | `true` / `false`（用于下拉搜索框）                                                                                        |

## 12. 十、可视化编辑器

用浏览器打开 `yaml_editor.html`，无需安装任何依赖。

### 模块管理

| 操作           | 方法                          |
| -------------- | ----------------------------- |
| 添加 JSON 文件 | 点击"添加 JSON"按钮或侧栏 `+` |
| 切换模块       | 点击左侧栏模块名              |
| 关闭模块       | 鼠标悬浮后点 ✕                |
| 搜索模块       | 侧栏顶部搜索框                |

### 编辑元素动作

1. 点击元素行的"编辑"按钮
2. 在弹窗中配置：
   - **操作类型**: click / fill / select / hover 等
   - **找不到时**: fail（终止）或 skip（跳过）
   - **填写内容**: fill 时的值，支持 `${变量名}`
   - **先点击再输入**: 用于下拉搜索框
   - **等待时间**: 操作前/后等待秒数
   - **等待策略**: timeout / navigation / element_appear
   - **IF 条件**: 条件执行（元素存在/不存在时）
3. 点击"保存"

### 导出 YAML

1. 文件名默认自动生成：`actions_config_{模块名}.yaml`
2. 可自定义文件名
3. 点击"导出 YAML"

### 变量管理

点击"变量"标签页切换：

1. 点击工具栏"加载 VARS"按钮选择 `vars.yaml`
2. 修改变量值
3. 点击"保存 VARS"覆盖原文件

## 附录

### 常见问题

**Q: 提示找不到浏览器实例？**
A: 确保先运行 `python start_browser.py` 或已打开 Chrome 并开启 CDP（端口 18800）。

**Q: 元素定位不到？**
A: 检查：

1. 是否在正确的页面上（URL 是否匹配）
2. 是否有弹窗遮挡
3. 尝试重新采集 + 清洗
4. 检查清理后的策略是否合理

**Q: 输入框找到但填不进去？**
A: 设置 `click_first: true`，先点击激活再输入。

**Q: 动态 ID 定位失败？**
A: 用 `selector_cleaner.py` 重新清洗，最新版本已支持 `el-popper-container` 等动态 ID 前缀匹配。

**Q: 如何跳过某个元素？**
A: 在动作配置中设置 `on_not_found: skip`。

**Q: 流程执行到一半断了？**
A: 浏览器保持打开，用 `n` 热替换新的 JSON 文件，或 `i` 插入缺失的步骤。
