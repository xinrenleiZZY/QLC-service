## 五、配置工具

### `configure_actions.py` ✅ 交互式配置

**功能**: 交互式 CLI 工具，逐一询问每个元素的操作类型/参数，生成 YAML
**用法**: `python configure_actions.py xxx_cleaned.json`
**输出**: `actions_config_xxx.yaml`

### `yaml_editor.html` ✅ 可视化编辑器（纯前端）

**功能**: 浏览器中运行的 GUI 编辑器

- **左侧栏**: 标签页切换（模块 / 变量）
- **模块页**: 加载多个 JSON，按 step_name 分组，可视化编辑每个元素的 action 配置
- **变量页**: 加载/编辑 `vars.yaml`，支持添加/删除/修改变量值
- **导出**: 自定义文件名，覆盖源文件
- **特性**: 10 秒无操作自动继续、条件执行（IF）、`click_first` 等

## 六、执行引擎

### `advanced_runner.py` ✅ 核心执行引擎

**功能**: 流程执行状态机 + 双引擎定位

- **状态机**: FINDING → EXECUTING → WAITING → RETRYING → HUMAN_HELP → SUCCESS/SKIP/FAIL
- **CDPEngine**: Playwright CDP Session + Pyppeteer（辅通道，Pyppeteer 未启用）
- **OCREngine**: 截图 → Tesseract OCR → 文字位置点击（最后一层兜底）
- **StepRunner**: 单个元素执行器，循环策略定位 → 执行动作 → 等待策略
- **配置模型**: `RetryConfig`, `HumanInterventionConfig`, `CDPConfig`
- **已实现**: `click_first` 先点击再输入

### `module_runner.py` ✅ 单模块执行器

**功能**: 执行一个 cleaned JSON 中的所有元素

- 兼容 3 种 JSON 格式（V10 包裹/展平/传统）
- 按 step_name 分组执行
- 自动加载 actions YAML 和 vars.yaml
- 智能默认动作（按 tag 判断 click/fill/select）
  **被**: `orchestrator.py` 调用

### `orchestrator.py` ✅ 流程编排器

**功能**: 多模块流程管理

- 加载 `flow.yaml` 定义
- 启动/复用浏览器
- 依次执行每个模块
- 失败时弹出交互菜单（r重试/a全部重来/h人工/n热替换/i插入/s跳过/q终止）
- 模块成功时 10 秒无操作自动继续
- **条件跳转**: 支持 `on_failure: {action: goto, target: xxx}` / `on_success` / `pre_flows`
- **热替换**: 运行时用新 JSON 替换当前模块
- **插入步骤**: 运行时插入新模块到流程
